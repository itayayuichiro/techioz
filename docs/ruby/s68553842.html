解決策

他の解決策 (sudo Arch -x86_64 gem install ffi) が機能しない場合は、代わりに次の解決策を試してください。

gem install –user-install ffi – –enable-lifffi-alloc

その後、pod install または実行しようとしていたものを再度実行しますが、先頭に Arch -x86_64 を付けずに実行します。

私にとっては問題なく動作し、この方法で Intel エミュレーション (Rosetta 2) の使用を避けることもできました。

この解決策は、ffi github プロジェクトに登録された問題で見つかりました。
