Fedora 38 での Ruby on Rails のインストールが機能しない

概要

Fedora 38 を実行していて、Ruby on Rails をインストールしようとしています。以前の試みでは、dnf 経由で Ruby をインストールしましたが、すべてが /usr/local/bin などに移動し、sudo として実行されていない権限エラーが発生したため、あまりうまくいきませんでした。

それをすべて削除して（おそらく）最初からやり直しましたが、今回はrbenvを使用しました。

Rails をインストールするとき、システムがどこにもそれを見つけられないことを除いて、すべてが機能しているようです。

gem install Rails -v 7.1.1 を実行すると、成功したことが表示されます。ただし、ターミナルに「rails」と入力するとエラーが発生します。見つかりません。

shimsフォルダー内を検索してみましたが、Railsがありません。

Ruby、gem、バンドルなどはすべて問題なく動作しているようです。 Rails だけが見つかりません。

私が何を間違っているのか知っていますか？

ありがとう

解決策

アップデート

おそらくRails 7.1.1は新しすぎるようです。 7.1.0 をインストールしたところ、問題なく動作しました。 7.1.1 がリリースされたばかりであることは知っていますが、RubyMine は今日早くそれを見つけました。しかし、何かが間違っています。

誰かが望むなら私の質問を削除することもできますが、おそらくこれは同じ問題を抱えている他の人を助けるかもしれません。
