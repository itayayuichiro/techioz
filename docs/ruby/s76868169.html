Spring Preloader を実稼働環境にインストールすべきでないのはなぜですか?

概要

私は Spring とは何か、そしてなぜアプリケーション プリローダーが必要なのかについていくつかの記事を読んでいました。ただし、アプリケーション プリローダーは開発をスピードアップし、開発環境で有効に使用できます。

私が理解したいのは、なぜ Spring Pre-loader を実稼働環境にインストールすべきではないのかということです。

これは、特に Ruby on Rails アプリケーションに関連します。

Googleでこれに関連する回答が見つかりません。

解決策

その README.md で履歴を確認してください。元の説明はこちらです。

この問題は、bundle exec Rails コンソールのプロダクションが実行されたときに Spring が開始されるという問題に起因しているようです。ブロックされている理由は、Rails Console を使用しているユーザーを驚かせないためです。詳細については、元の問題を参照してください
