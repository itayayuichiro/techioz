概要

最近、プロジェクトを Ruby 3.2.2 と Rails 6.1 にアップグレードしました。 ActionView::Base で問題が発生しています。最近のアップグレードでは、ActionView::Base.new に 2 つの引数ではなく 3 つの引数を渡す必要があります。ドキュメントによると、ActionView は lookup_context、assigns、controller を受け取るようになりました。

view = ActionView::Base.new(
  ActionController::Base.view_paths, 
 {
  client: client,  
  report: self.report, 
},
nil
)

アクションビューのバージョンは6.1.7.3です。

解決策

Rails 6.1 では、ActionView::Base.new は 3 つの引数を取ります: lookup_context、assigns、controller

ただし、ActionView::LookupContext の代わりに ActionView::PathSet インスタンスを渡しました。

このオブジェクトにはそのようなゲッターがないため、エラーが発生します

このラップパスの代わりに、ルックアップコンテキストにパスを設定します。また、新しいRubyでは、名前がキー名と等しい場合、重複する値は必要ありません

view =
  ActionView::Base.new(
    ActionView::LookupContext.new(ActionController::Base.view_paths),
    { client:, report: },
    nil,
  )

その後、このビューをレンダリングに使用できます
