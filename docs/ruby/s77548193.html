最初に /bin/bash –login コマンドを実行しないと、新しい Rails アプリを作成できません。

概要

私の Ruby は 3.0.2 で、Ruby のバージョンを 3.2.2 にアップグレードしましたが、新しい Rails アプリを作成しようとすると、次のエラーが発生します。 /usr/bin/env: ‘ruby3.0’: そのようなファイルまたはディレクトリはありません したがって、Rails アプリを作成して RVM にアクセスするたびに、「/bin/bash –login」を実行します。最初に /bin/bash –login コマンドを使用せずに、同じ古い方法で Rails アプリを作成したいと考えています。私は何をすればいいのでしょうか？

RVM は 3.2.2 –default を使用します

解決策

「ログインシェル」を使用するには端末を変更する必要があります。これは、使用している OS / ターミナル エミュレータによって動作が異なります。

iTerm2 を使用している MacOS では、[プロファイル] -> [プロファイルを開く] に移動する必要があります。使用しているプロファイルを選択して、プロファイルを編集し、コマンドを「ログイン シェル」に変更します。

他のシェルエミュレータでこれを行う方法はわかりませんが、これで始めることができるはずです。
