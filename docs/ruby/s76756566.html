米国 A2P 10DLC 個人事業主のブランド、キャンペーン、およびメッセージング サービスの検証にもかかわらず、テキストが引き続きエラー: 30035 でフィルタリングされる

概要

Twilio で送信された Ruby on Rails アプリからのテキストはフィルターされ続けます。私の小さなアプリは、スケジュール情報を含むテキストを毎晩約 50 通送信します。私は 2023 年 5 月に Twilio サイト ブランドに個人事業主を登録しました。認証済みのキャンペーンとメッセージング サービスを利用しており、メッセージを送信する番号は 1 つだけです。 https://console.twilio.com/us1/develop/sms/try-it-out/send-an-sms のコードを使用して 15% と 25% の間のフィルタリングをテストするとブロックされます。何かアイデアがある人はいますか？

私のテストコードは次のとおりです。

require 'rubygems'
require 'twilio-ruby'

account_sid = ENV['TWILIO_ACCOUNT_SID']
auth_token = ENV['TWILIO_AUTH_TOKEN']
@client = Twilio::REST::Client.new(account_sid, auth_token)

n=0
20.times do
  message = @client.messages.create(
    #body: "number test from #{n}",
    body: "messaging service test from #{n}",
    messaging_service_sid: ENV['MESSAGING_SERVICE_SID'],
    #from: ENV['TWILIO_APP_NUMBER'],
    to: ENV['MY_CELL_NUMBER']
    )

    puts message.sid
    n+=1
end

20 個のテスト メッセージのうち 3 ～ 5 個 (15 ～ 25%) が、「(エラー: 30035) US A2P 10DLC - 設定中の番号からのメッセージ」エラーでブロックされます。

このアマチュアについて何かご意見がございましたら、よろしくお願いいたします。

解決策

Twilio サポートから折り返し連絡があり、メッセージング サービスから自分の番号を削除して、再度追加する必要があることがわかりました。手順は https://support.twilio.com/hc/en-us/articles/4402705042075-Managing-a-Messaging-Service-Sender-Pool にあります。

メッセージング サービスをオフにして、再度オンにする必要がありました。
