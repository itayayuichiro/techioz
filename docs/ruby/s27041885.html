RVM のインストール中に「gpg: コマンドが見つかりません」エラーを解決するにはどうすればよいですか?

概要

新しい Mac Pro (OS X 10.9.5) を持っているので、最初からセットアップします。 RVM をインストールしたいのですが、最初に次のように指示されます。

試してみたところ、次のようになりました。

gpg --keyserver hkp://keys.gnupg.net --recv-keys D39DC0E3
zsh: command not found: gpg

これを克服する方法について、信頼できそうな良いガイドを見つけようとしましたが、うまくいきませんでした。

GPG とは何なのか、なぜ私が GPG を持っていないのか、そしてどのようにすれば正しい方法で入手できるのかを誰かが説明してもらえますか。

解決策

GnuPG (バイナリ名 gpg) は、OpenPGP プロトコルを使用した公開キー暗号化に使用されるアプリケーションですが、署名 (暗号署名、正しく使用されれば発行者の検証も可能) の検証にも使用されます。ある意味、これは OpenSSL が X.509 および TLS に相当するのと同じ、OpenPGP に相当すると言えるでしょう。

ほとんどの Linux ディストリビューション (パッケージ リポジトリ内でソフトウェアが改ざんされていないことを保証するために GnuPG を多用する) とは異なり、Mac OS X にはオペレーティング システムに GnuPG が付属していないため、自分でインストールする必要があります。
