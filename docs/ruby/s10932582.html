「/パス/へ」とは何ですか?

概要

私はしばらくの間 Web プログラミングを行っており、自分自身をまったくの初心者とは思っていませんが、「/path/to」が何であるかをまだ理解していません。人々はそのコードを頻繁に使用しますが、私はこれがメインパスを参照するための単なる方法だと思っていました。しかし、私は、なぜ多くの人がその構文をこれほど一律に使用するのか疑問に思い始めました。それは、文字通りに解釈することを意図していない場合、混乱を招くからです。ほとんどの人は実際に「/path/to」と入力するでしょう。

そこで Google で「/path/to」を検索してみましたが、これは一般的な検索エンジンでは検索するのが難しいため、うまくいきませんでした。そこで、ここで質問させていただくことにしました。 「/path/to」は、人々が何かを指すために使用するある種の専門用語ですか? 「はい」の場合、それは正確に何を指しますか? 「いいえ」の場合、内部的にはどのように動作するのでしょうか?

解決策

これは、環境内の実際のパスの単なるプレースホルダーです。私は通常、自分のマシンで使用するパスから抽象化したいときにこれを使用します。それは重要ではなく、私のコードの読者/ユーザーはおそらくそれを異なるものと考えるでしょう。したがって、どの箇所を修正する必要があるのかを明確に示すことを好みます。

この種の他の例:

GET http://example.com
ssh username@host
