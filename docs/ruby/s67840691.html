ld: macOS Big Sur 11.4 Apple M1 上の mysql2 gem Ruby のバンドルインストール中に -lzstd のライブラリが見つかりません

概要

バンドルインストールの実行中

An error occurred while installing mysql2 (0.5.3), and Bundler cannot continue.
Make sure that `gem install mysql2 -v '0.5.3' --source 'https://rubygems.org/'` succeeds before bundling.

Rails アプリ 6.1.3

バンドラー バージョン 2.2.19

macOS ビッグサー 11.4

アップル M1

Gemfile 内

ruby '2.6.6'
gem 'rails', '~> 6.1.3', '>= 6.1.3.1'
gem 'mysql2'

Gemfile.lock 内の mysql は 0.5.3 です

brew install mysql はすでに正常に実行されています…もう一度実行すると、次のように表示されます。

警告: mysql 8.0.25_1 はすでにインストールされており、最新の状態です。 8.0.25_1 を再インストールするには、次のコマンドを実行します。 醸造、mysql の再インストール

解決策

ステップ1：

openssl と MySQL の両方が brew 経由でインストールされていることを確認します

mysqlを醸造インストールする

醸造インストールopenssl

または、openssl の以前のバージョンをお持ちの場合は、試してみてください

醸造インストール openssl@3

ステップ2：

Rails アプリでこれを実行して、バンドラーを通過できることを確認します。

gem install mysql2 -v '0.5.3' -- --with-opt-dir=$(brew --prefix openssl) --with-ldflags=-L/opt/homebrew/opt/zstd/lib

注意すべき 2 つの点: /usr/local/opt/openssl@3 は、評価内の内容をマシン上の openssl ディレクトリに動的に置き換えます。

/opt/homebrew/opt/zstd は実際には、マシン上の zstd の現在のバージョンへのシンボリックリンクです (/opt/homebrew/opt/Cellar/zstd/1.5.2 のようなものにリンクします)。

また、0.5.3 は mysql2 の最新の gem バージョンではなくなったことにも注意してください。最新バージョンはここで見つけてください。

https://translate.google.com/translate?hl=ja&sl=en&tl=ja&u=https://rubygems.org/gems/mysql2

ステップ 3:

これをグローバルに実行したい場合は、これを実行します。

bundle config --global build.mysql2 "--with-opt-dir=$(brew --prefix openssl) --with-ldflags=-L/opt/homebrew/opt/zstd/lib"

これにより ~/.bundler/config ファイルが変更され、バンドラーにグローバルに適用されることに注意してください。
