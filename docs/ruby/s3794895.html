Ruby サポート (+ruby) を使用して vim をインストールする

概要

vim に command-t をインストールしようとしていますが、現在のバージョンの vim には (+ruby) フラグがありません。 「どのruby」というコマンドは、rubyがインストールされていることを示します。

vim で +ruby フラグを有効にするには何をする必要がありますか?

また、+ruby フラグは技術的に何を意味しますか?

解決策

一部のパッケージは、vim-nox など、Ubuntu で vim-ruby を提供します。

単に：

sudo apt install vim-nox

これは、Ruby を使用した vim を取得するだけでなく、「Perl、Python、Ruby、および TCL によるスクリプト作成をサポートしますが、GUI はサポートしません」でコンパイルされます。

sudo apt install vim-rails

「Ruby on Rails アプリケーションの編集を非常に簡単にする一連の vimscript」をインストールします。しかし、これは vim-full と vim-addon-manager に依存しているため、vim-gnome バージョンの GUI もインストールされます。つまり、「GNOME2 GUI でコンパイルされた vim のバージョンで、Perl、Python、Ruby によるスクリプト作成がサポートされています」 、TCL。」
