rubocop マジックコメントの欠落を修正するにはどうすればよいですか

概要

Ruby コードのライブラリがあり、欠陥を探すために実行します。

$ rubocop

そして、私は得ます

$ rubocop
Inspecting 153 files
CCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCWCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCCWCCCCCCCWWCCCCC

Offenses:

Gemfile:1:1: C: Missing magic comment # frozen_string_literal: true.
source "https://rubygems.org"

rubocop が文句を言わないようにするために、Gemfile にどのような変更が必要ですか?

解決策

追加するだけです

# frozen_string_literal: true

各 Ruby ファイルの最初の行に。あるいは走る

rubocop -a

Rubocop が修正可能なすべての違反を自動的に修正できるようにします。

ところで。私は Rubocop が好きで自分でも使っていますが、それが検出したものを欠陥とは呼びません。私はこのリストを、同僚との議論の提案や理由のように考えています。
